# Falling Snack Game Plan

현재 버전의 방향은 `하늘에서 떨어지는 간식을 먹어 최고 점수를 만들고, 최고 점수만 랭킹에 남기는 웹 게임`이다.

## 목적

- 주어진 배경과 캐릭터 스프라이트를 사용해 바로 플레이 가능한 2D 게임을 만든다.
- 짧은 한 판 안에서 점수 경쟁이 분명하게 느껴지도록 한다.
- 점수는 서버에 저장하고, 같은 닉네임은 최고 점수만 유지한다.

## 자산 매핑

- `scene/c.png`: 기본 게임 배경
- `scene/5.png`: 게임오버 장면
- `character/0.png`: 서 있기
- `character/1.png`: 걷기 1
- `character/2.png`: 걷기 2
- `character/3.png`: 점프
- `character/4.png`: 데미지
- `item/01.png`: 점수 아이템 1
- `item/02.png`: 점수 아이템 2
- `item/03.png`: 점수 아이템 3
- `item/04.png`: 점수 아이템 4
- `item/001.png`: 감점 아이템 1
- `item/002.png`: 감점 아이템 2
- `item/003.png`: 감점 아이템 3

## 코어 플레이

1. 플레이어가 닉네임을 입력하고 게임을 시작한다.
2. 1라운드는 `scene/c.png` 배경에서 시작하고, 초반 아이템은 느리게 떨어진다.
3. `01~04` 아이템은 점수를 올린다.
4. `001~003` 아이템은 점수를 깎고 `character/4.png` 데미지 장면을 재생한다.
5. 시간이 지나면 2라운드 `scene/c2.png` 배경으로 넘어가고 낙하 속도와 생성 속도가 더 빨라진다.
6. 제한 시간이 끝나면 `scene/5.png` 게임오버 장면과 최종 점수, 랭킹이 표시된다.
7. 점수는 서버에 저장되고 아래 실시간 랭킹에 반영된다.

## 규칙 기준

- 기본 플레이 시간: `60초`
- 라운드 전환: `30초` 시점에 2라운드 진입
- 입력: `좌우 이동`, `점프`
- HUD: 게임 화면 내부에 `점수`, `남은 시간`
- 랭킹: 화면 아래 별도 패널 유지
- 저장 정책: 같은 닉네임은 최고 점수만 남김
- 감점 점수: 현재 `-20`, `-26`, `-32`

## 현재 범위

- 웹 브라우저에서 바로 실행되는 싱글 플레이
- 서버 저장형 온라인 랭킹
- 실시간 대전 멀티플레이는 현재 범위 밖

## 리스크

- 캐릭터 스프라이트 원본 비율에 따라 화면에서 너무 마르거나 넓어 보일 수 있다.
- 아이템 크기와 충돌 반경이 맞지 않으면 먹는 느낌이 어색해질 수 있다.
- 랭킹은 최고 점수 저장형이라 동시 접속 경쟁전과는 구조가 다르다.
